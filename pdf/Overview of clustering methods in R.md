## Что такое кластеризация?

**Кластерный анализ** или кластеризация — это задача группировки набора объектов таким образом, чтобы объекты в одной группе (называемой кластером) были более похожи (в том или ином смысле) друг на друга, чем на объекты в других группах (кластерах).

Это метод **обучения без учителя**, поэтому кластеризация используется, когда нет априорной информации о данных.
Это делает кластеризацию очень сильным методом для получения понимания данных и принятия более точных решений.

### Для чего это нужно?

Кластеризация используется для:

*   Получения понимания данных, генерации гипотез, обнаружения аномалий и выявления характерных признаков,
*   Определения степени сходства между объектами (т.е. организмами),
*   В качестве метода организации данных и их обобщения с помощью прототипов кластеров (сжатие).

## Типы методов кластеризации

Поскольку кластерный анализ существует уже более 50 лет, имеется большое количество доступных методов.
Базовая классификация методов кластеризации основана на цели, к которой они стремятся: **иерархические**, **неиерархические**.

**Иерархическая кластеризация** — это многоуровневое разбиение набора данных, которое является ветвью классификации (кластеризации). Иерархическая кластеризация имеет два типа доступа к данным. Первый, дивизивная (разделяющая) кластеризация, начинается с одного большого кластера, который затем делится на более мелкие кластеры. Второй, агломеративная (объединяющая) кластеризация, начинается с отдельных объектов, которые являются одноэлементными кластерами, а затем они постепенно объединяются. Весь процесс иерархической кластеризации может быть выражен (визуализирован) в виде дендрограммы.

**Неиерархическая кластеризация** — это разделение набора данных на систему дизъюнктивных подмножеств (кластеров) таким образом, чтобы пересечение кластеров было пустым множеством.

Методы кластеризации также можно разделить более подробно на основе процессов в самом методе (алгоритме):

**Неиерархические**:

*   На основе центроидов (Centroid-based)
*   На основе моделей (Model-based)
*   На основе плотности (Density-based)
*   На основе сетки (Grid-based)

**Иерархические**:

*   Агломеративные (Agglomerative)
*   Дивизивные (Divisive)

Но какой из них выбрать в вашем случае? Давайте углубимся в наиболее известные методы и обсудим их преимущества и недостатки.

### На основе центроидов

Самый базовый (возможно, только для меня) тип метода кластеризации — на основе центроидов.
Этот тип кластеризации создает прототипы кластеров - центроиды или медоиды.

Наиболее известные методы:

*   K-средних (K-means)
*   K-медиан (K-medians)
*   K-медоидов (K-medoids)
*   K-мод (K-modes)

#### K-средних (K-means)

Шаги:

*   Создать случайные K кластеров (и вычислить центроиды).
*   Назначить точки ближайшим центроидам.
*   Обновить центроиды.
*   Переходить к шагу 2, пока центроиды изменяются.

Плюсы и минусы:

*   [+] Быстро вычисляется. Легко понять.
*   [-] Различные начальные кластеры могут привести к разной итоговой кластеризации.
*   [-] Зависит от масштаба.
*   [-] Создает только выпуклые (сферические) формы кластеров.
*   [-] Чувствителен к выбросам.

**K-средних - пример**

Очень легко попробовать K-средних в R (с помощью функции `kmeans`), единственный необходимый параметр — это количество кластеров.

```r
km_res <- kmeans(data_example, 3)$cluster
 
data_example[, class := as.factor(km_res)]
 
centroids <- data_example[, .(x = mean(x), y = mean(y)), by = class]
 
ggplot(data_example, aes(x, y, color = class, shape = class)) +
  geom_point(alpha = 0.75, size = 8) +
  geom_point(data = centroids, aes(x, y), color = "black", shape = "+", size = 18) +
  theme_bw()
```

Мы можем видеть пример, когда K-средних чаще всего дает сбой, то есть когда в наборе данных есть выбросы.

#### K-медоидов (K-medoids)

Проблему с выбросами решает метод K-медоидов, поскольку прототипами являются медоиды — члены набора данных. То есть не искусственно созданные центроиды, что помогает справляться с выбросами.

Плюсы и минусы:

*   [+] Легко понять.
*   [+] Менее чувствителен к выбросам.
*   [+] Возможность использовать любую меру расстояния.
*   [-] Различные начальные кластеры могут привести к разной итоговой кластеризации.
*   [-] Зависит от масштаба.
*   [-] Медленнее, чем K-средних.

**K-медоидов - пример**

Проблема K-медоидов может быть решена с помощью алгоритма Partition Around Medoids (PAM) (функция `pam` в пакете `cluster`).

```r
kmed_res <- pam(data_example[, .(x, y)], 3)
data_example[, class := as.factor(kmed_res$clustering)]
 
medoids <- data.table(kmed_res$medoids, class = as.factor(1:3))
 
ggplot(data_example, aes(x, y, color = class, shape = class)) +
  geom_point(alpha = 0.75, size = 8) +
  geom_point(data = medoids, aes(x, y, shape = class),
             color = "black", size = 11, alpha = 0.7) +
  theme_bw() +
  guides(shape = "none")
```

Мы видим, что медоиды хорошо расположились в трех основных группах данных.

#### Определение количества кластеров

Недостатком методов на основе центроидов является то, что количество кластеров должно быть известно заранее (это параметр методов). Однако мы можем определить количество кластеров с помощью внутренней валидации (индекса). Основные шаги основаны на том, что мы вычисляем некоторый индекс внутренней валидации с множеством ( K ) и выбираем ( K ) с наилучшим значением индекса.

Существует множество индексов…

*   Силуэт (Silhouette)
*   Индекс Дэвиса-Болдина (Davies-Bouldin index)
*   Индекс Данна (Dunn index)
*   и т.д.

Однако каждый индекс имеет схожие характеристики:

$$ \frac{within-cluster-similarity}{between-clusters-similarity} . $$

то есть это отношение средних расстояний внутри кластеров и между кластерами.

#### Диаграмма локтя (Elbow diagram)

Диаграмма локтя — это простой метод (правило) определения количества кластеров: мы вычисляем внутренний индекс для набора K и выбираем K, где положительное изменение является наибольшим.

Так, например, я выбрал индекс Дэвиса-Болдина, реализованный в пакете `clusterCrit`. Для нашего простого набора данных я сгенерирую кластеризации с количеством кластеров от 2 до 6 и вычислю индекс.

### На основе моделей

Методы кластеризации на основе моделей базируются на некотором вероятностном распределении. Это может быть:

*   Гауссовское нормальное распределение
*   Гамма-распределение
*   t-распределение Стьюдента
*   Распределение Пуассона
*   и т.д.

Поскольку мы кластеризуем многомерные данные, кластеризация на основе моделей использует многомерные распределения и так называемую смесь моделей (Смеси -> кластеры). При использовании кластеризации с гауссовским нормальным распределением мы используем теорию **моделей гауссовской смеси (GMM)**.

#### GMM

Цель состоит в максимизации правдоподобия:
$$ L(\boldsymbol{\mu_1}, \dots, \boldsymbol{\mu_k}, \boldsymbol{\Sigma_1}, \dots, \boldsymbol{\Sigma_k} | \boldsymbol{x_1}, \dots, \boldsymbol{x_n}). $$
Здесь кластер представлен средним значением (( \mathbf{\mu} )) и ковариационной матрицей (( \mathbf{\Sigma} )). То есть не просто центроидом, как в случае K-средних.

Эта задача оптимизации обычно решается с помощью алгоритма **EM** (Expectation Maximization - максимизация ожидания).

Плюсы и минусы:

*   [+] Эллипсоидальные кластеры,
*   [+] Может быть параметризован ковариационной матрицей,
*   [+] Не зависит от масштаба,
*   [-] Очень медленно для многомерных данных,
*   [-] Может быть сложным для понимания.

Алгоритм EM с GMM реализован в пакете `mclust`. Вы можете оптимизировать различные формы смесей (кластеров) с помощью параметра `modelNames` (проверьте функцию `?mclustModelNames` для получения более подробной информации).

```r
library(mclust)
res <- Mclust(data_example[, .(x, y)], G = 3, modelNames = "VVV", verbose = FALSE)
 
plot(res, what = "classification")
```

Довольно интересный красный эллипс, который был создан, но в целом кластеризация нормальная.

В пакете `mclust` вы можете просто добавить несколько `modelNames`, и он выберет по BIC наилучший.

### На основе плотности

Кластеры на основе плотности базируются на максимально связных компонентах множества точек, которые лежат в пределах некоторого определенного расстояния от некоторого корневого (core) объекта.

Методы:

*   DBSCAN
*   OPTICS
*   HDBSCAN
*   Методы множественной плотности (Multi-density)

#### DBSCAN

В хорошо известном методе DBSCAN плотность определяется как *окрестность*, где точки должны быть достижимы в пределах определенного расстояния (расстояние ( \epsilon ) - первый параметр метода), однако кластеры должны иметь по крайней мере некоторое минимальное количество точек (второй параметр метода). Точки, которые не были связаны ни с одним кластером и не прошли критерий минимального количества точек, помечаются как шум (выбросы).

Плюсы и минусы:

*   [+] Автоматически извлекает выбросы,
*   [+] Быстро вычисляется,
*   [+] Может находить кластеры произвольной формы,
*   [+] Количество кластеров определяется автоматически на основе данных,
*   [-] Параметры (( \epsilon ), minPts) должны быть заданы практиком,
*   [-] Возможная проблема с окрестностями - они могут быть связаны.

DBSCAN реализован в одноименной функции и пакете, так что давайте попробуем.

```r
library(dbscan)
 
res <- dbscan(data_example[, .(x, y)], eps = 0.4, minPts = 5)
table(res$cluster)
```

### Спектральная кластеризация

Методы спектральной кластеризации основаны на спектральном разложении данных, то есть на создании собственных векторов и собственных значений.

Шаги:

1.  N = количество данных, d = размерность данных,
2.  ( \mathbf{A} ) = матрица аффинности (сходства), ( A_{ij} = \exp(- (data_i - data_j)^2 / (2*\sigma^2) ) ) - матрица N на N,
3.  ( \mathbf{D} ) = диагональная матрица, чей (i,i)-й элемент является суммой i-й строки ( \mathbf{A} ) - матрица N на N,
4.  ( \mathbf{L} ) = ( \mathbf{D}^{-1/2} \mathbf{A} \mathbf{D}^{-1/2} ) - матрица N на N,
5.  ( \mathbf{X} ) = объединение k наибольших собственных векторов ( \mathbf{L} ) - матрица N на k,
6.  Перенормировка каждой из строк ( \mathbf{X} ) для получения единичной длины - матрица N на k,
7.  Запуск алгоритма K-средних на ( \mathbf{X} ).

### Иерархическая кластеризация

Результатом иерархической кластеризации является дендрограмма. Дендрограмму можно разрезать на любой высоте, чтобы сформировать разбиение данных на кластеры.
То, как точки данных соединяются в дендрограмме, имеет множество возможных способов (связей) и критериев:

*   Одиночная связь (Single-linkage)
*   Полная связь (Complete-linkage)
*   Средняя связь (Average-linkage)
*   Центроидная связь (Centroid-linkage)
*   Метод минимальной дисперсии Уорда
*   и т.д.

Критерии:

*   одиночная связь: ( \min \{ d(a,b):a\in A, b\in B \} )
*   полная связь: ( \max \{ d(a,b):a\in A, b\in B \} )
*   средняя связь: ( \frac{1}{|A||B|}\sum_{a\in A}\sum_{b\in B}d(a,b) )
*   центроидная связь: ( ||c_t - c_s|| ), где ( c_s ) и ( c_t ) — центроиды кластеров ( s ) и ( t ).

где ( d(a,b) ) — расстояние между точками ( a ) и ( b ).

### Другие типы методов кластеризации

## Выводы

*   Существует множество типов методов кластеризации
*   Разные наборы данных требуют разных методов кластеризации
*   Автоматическое определение количества кластеров может быть сложным
*   Реальные наборы данных обычно связаны - методы на основе плотности могут дать сбой
*   Выбросы (аномалии) могут существенно повлиять на результаты кластеризации - решением является предварительная обработка данных или использование кластеризации на основе плотности
*   Некоторые методы не подходят для больших (или многомерных) наборов данных - кластеризация на основе моделей или спектральная кластеризация
*   Некоторые методы не подходят для невыпуклых кластеров - K-средних, базовая иерархическая кластеризация
